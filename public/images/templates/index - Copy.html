🎨 CSS POSITIONING EXAMPLES

Loading sprites with CSS

.sprite {
  width: 256px;
  height: 512px;
  background-image: url('/images/sprites/characters.png');
  background-repeat: no-repeat;
}

/* Position sprites using background-position */
.char-1-1 { background-position: 0 0; }
.char-1-2 { background-position: -256px 0; }
.char-1-3 { background-position: -512px 0; }
.char-1-4 { background-position: -768px 0; }
.char-1-5 { background-position: -1024px 0; }
.char-1-6 { background-position: -1280px 0; }

.char-2-1 { background-position: 0 -512px; }
.char-2-2 { background-position: -256px -512px; }
/* ... etc */

HTML Usage

<div class="character-card">
  <div class="sprite char-2-3" style="background-position: -512px -512px;"></div>
  <span class="name">My Character</span>
</div>
✅ FILE STRUCTURE

When you create your spritesheets, place them in:

public/images/sprites/characters.png
public/images/sprites/buddies.png
public/images/sprites/ui.png

📝 NAMING CONVENTION

  • characters.png - Main character sprites (12 slots)
  • buddies.png - Companion sprites (12 slots)
  • ui.png - UI elements (2048×2048)

🔧 TECHNICAL NOTES

  • All spritesheets should be PNG format
  • Use transparent backgrounds where appropriate
  • Characters: 256×512 per cell, 72 DPI recommended
  • UI: Can be any size but 2048×2048 is optimal for caching
  • For best performance, spritesheets should be power-of-2 dimensions
